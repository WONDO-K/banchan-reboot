# Cursor 기반 코드 개선 프로젝트

## 1. 프로젝트 개요

본 저장소는 기존 팀 프로젝트를 기반으로,  
AI 기반 개발 도구(Cursor)를 활용한 코드 개선 작업을 진행하기 위한 개인 저장소이다.

현재는 개선 작업 전 초기 상태이며,  
향후 리팩토링 및 DB 성능 개선 과정을 단계적으로 정리할 예정이다.

---

## 2. 저장소 성격

- 기존 팀 프로젝트를 기반으로 한 분리된 개선 작업용 저장소
- 연속 개발 목적이 아닌, 코드 구조 및 성능 개선을 위한 관리 공간
- 변경 결과보다 개선 과정과 판단 기준 기록에 중점

---

## 3. 개선 진행 방향 (예정)

- Cursor를 활용한 코드 리팩토링
- 기존 코드 구조에 대한 점진적 개선
- DB 접근 방식 및 성능 관점 개선 가능성 검토

---

## 4. 개선 진행 방식

- 파일 단위 또는 서비스 단위로 작업 진행
- 한 번에 하나의 개선 작업만 수행
- 변경 전 상태를 기준점으로 비교
- 모든 변경 사항은 커밋 단위로 기록

---

## 5. 개선 작업 기록 공간

> 향후 개선 작업이 아래 영역에 누적될 예정이다.
ex)
- 리팩토링 내역
- DB 성능 개선 내역
- Cursor 활용 코드 개선 사례

---

## 6. 기술 스택 (기준 프로젝트)

- Backend: Java, Spring Boot
- Database: MySQL

---

## 7. 향후 정리 예정

- 개선 단위별 변경 요약
- 리팩토링 및 성능 개선 판단 기준 정리
- 포트폴리오용 설명 정리
